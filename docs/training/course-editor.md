# 课程编辑器（Course Editor）

## 一、模块定位（What \u0026 Why）

### 1. 核心定位

**课程编辑器（Course Editor）** 是商家端训练计划中的核心生产工具，用于：

\u003e 将"原子动作"按训练逻辑，编排为一节可执行、可计量、可复用的训练课程

它解决的不是「写训练内容」，而是 **控制训练节奏与容量**。

### 2. 结构抽象（三层模型）

训练计划采用 **三级结构模型**：

\u0060\u0060\u0060
训练计划（Plan）
 └── 课程（Course） × N
      └── 训练阶段（Stage） × N
           └── 动作（Exercise） × N
\u0060\u0060\u0060

- **动作（Exercise）** 是系统中的最小原子单位，不可再拆

---

## 二、使用场景（When）

课程编辑器主要服务于以下场景：

- 商家 **编排标准课程**
- 为会员定制 **个人训练计划**
- 构建 **可复用课程模板**

---

## 三、页面整体结构（Layout）

页面采用 **「头部固定 + 内容滚动 + 底部固定」** 结构：

### 1️⃣ 顶部导航（Sticky Header）

| 元素 | 说明 |
|------|------|
| 返回按钮 | 退出编辑 |
| 页面标题 | 编辑课程 |

\u003e 不承载复杂操作，保证编辑时的安全感与退出路径

### 2️⃣ 内容区（Scrollable Area）

- 由多个 **训练阶段模块（StageSection）** 组成
- 阶段之间为 **纵向堆叠结构**
- 底部提供 **「新增训练阶段」** 占位入口

### 3️⃣ 底部操作区（Sticky Footer）

唯一主操作按钮：**确认并保存编排**

\u003e 这是一个编辑型页面，不鼓励多动作并行

---

## 四、训练阶段模块（StageSection）—— 核心结构

每一个 **StageSection** 是一个完整、可复制、可删除的训练单元。

### A. 阶段头部（Stage Header）

| 内容 | 操作 |
|------|------|
| 阶段名称（如：热身准备 / 主要训练 / 放松拉伸） | **复制**：克隆当前阶段的容量配置 + 动作编排 |
| | **删除**：移除整个阶段（含所有动作） |

\u003e 阶段是一个 **逻辑闭环单位**，不支持拆分复制

### B. 训练容量配置（Capacity Configuration）【核心】

这是课程编辑器中最重要的控制区，用于定义 **这一阶段"怎么练、多大量"**。

#### 1️⃣ 模式切换：计时 / 计次

- 右上角 Toggle
- 两种模式互斥

\u003e 这是一个**全阶段级别**的配置，不作用于单动作

#### 2️⃣ 单动作练习（Per-Exercise Load）

**定义**：该阶段中，每一个动作的基础负荷

| 模式 | 示例 |
|------|------|
| 计时模式 | 45 秒 |
| 计次模式 | 12 次 |

#### 3️⃣ 循环组休息（Rest Between Sets）

**定义**：每一轮动作循环完成后的休息时间
- 固定单位：秒

#### 4️⃣ 重复方式（Repeat Pattern）

当前支持：

| 方式 | 说明 |
|------|------|
| **循环训练** | 通过依次完成一系列不同的训练动作来提高身体的耐力和力量。例如：俯卧撑 → 深蹲 → 仰卧起坐 → 回到俯卧撑，不断循环 |
| **分化训练** | 针对特定肌肉群或身体部位进行的训练。例如：加强胸肌，可进行卧推、平板哑铃飞鸟、俯卧撑等针对性动作 |

#### 5️⃣ 重复组数（Total Sets）

**定义**：整个阶段循环执行的次数
- 与「重复方式」组合决定真实训练容量

---

### 📊 容量模型总结

\u0060\u0060\u0060
一个阶段的真实训练量 = 单动作负荷 × 动作数 × 重复组数
\u0060\u0060\u0060

---

## 五、动作编排模块（Exercise Arrangement）

### 1. 模块结构

**标题区**
- 标题：动作编排
- 数量角标：当前阶段动作数（实时更新）

### 2. 动作列表（ExerciseItem）

每个动作项包含：

| 元素 | 说明 |
|------|------|
| 排序手柄 | 拖拽排序能力 |
| 动作封面 | 标准动作演示图 |
| 动作名称 | |
| 删除按钮 | 移除该动作 |

\u003e **动作顺序 = 实际训练执行顺序**

### 3. 添加动作

底部蓝色虚线按钮：**+ 添加训练动作**
- 点击后进入 **动作库选择流程**

---

## 六、关键设计原则（给 AI / 研发看的）

### 1️⃣ 动作是最小原子

- 所有训练逻辑都围绕 **动作组合**
- 不允许在动作层以下再做拆分

### 2️⃣ 阶段是"容量控制单位"

- 负荷、休息、循环都作用于 **阶段**
- 动作只负责"是什么"，阶段负责"怎么练"

### 3️⃣ 单一主行动

- 编辑页面只有一个最终动作：**保存**
- 防止训练结构在中途被打断
